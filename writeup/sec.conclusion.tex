Kritische Würdigung
\begin{itemize}
\item Threadproblematik (und damit verbundene Limitation bei den Experimenten -> Zeit und Entitätszahl))
\item Andere Einflüsse wie Medikamentation oder genetische Diversität / Immunisierung unberücksichtigt
\item Konfigurationen mit Strategie nach fest vorgegebenem Schema
\end{itemize}

Das benutzte Framework erzeugt für jede Entität einen neuen Thread. Die Java Virtual Machine reserviert für jeden Thread eine bestimmte Speichermenge. So kommt bei ausreichend große Anzahl von Entitäten dazu, dass zu wenig Arbeitsspeicher vorhanden ist. Deshalb werden Teile auf die Festplatte ausgelagert, wodurch sich die Dauer einer Simulation stark vergrößert. \textbf{Gibt es Medizin speziell gegen ABPV??? Betty?}. Ein Imker könnte nach dem Bemerken einer Erkrankung seiner Bienen ein Medikament verabreichen, um einer weiteren Ausbreitung vorzubeugen. In der Natur besitzen manche Bienenstämme eine natürliche Immunität gegen das Virus oder können diese nach einer Infektion entwickeln. Diese Aspekte wurden in diesem Projekt nicht betrachtet. So könnte nach einer Ansteckung noch entschieden werden, ob eine Biene auch erkrankt. Desweiteren wäre es möglich die Bienen einzelner Bienenstöcke immun gegen die Krankheit zu machen. Bisher werden die Bienenstöcke nach einem festen Schema platziert: Es gibt Gruppen von Stöcken, die mit größtmöglicher Entfernung untereinander platziert werden. Innerhalb einer Gruppe werden die Stöcke nah nebeneinander gesetzt. Dabei werden andere Anordnungen nicht simuliert. So sollten auch weitere geometrische oder zufällig gewählte Platzierungen überprüft werden.

Ausblick / Erweiterungsmöglichkeiten / \textbf{oben mit abhandeln?}
\begin{itemize}
\item Einbezug von Immunisierung oder genetischer Diversität
\item genauere Untersuchung des Infektionsverlaufs (nicht nur Zeit bis alle tot sind sondern auch lokale Ausbreitungsgeschwindigkeit z.B.)
\end{itemize}
